Online Pool Game (8 Ball Inspired)

This project is a multiplayer pool game built entirely from scratch, inspired by Miniclip’s 8 Ball Pool. I was responsible for the full gameplay, physics, and networking implementation.

Custom Physics System

After initially testing Unity’s built-in physics system, I realized it did not provide the level of precision and consistency required for a competitive pool experience. Following research and experimentation, I decided to implement a custom 2D physics solution tailored specifically for billiards gameplay.

The physics system simulates core principles such as:

  • Elastic collisions between balls
  • Inertia and velocity preservation
  • Friction and gradual deceleration
  • Accurate ball-to-ball and ball-to-wall interactions

The result was a deterministic and consistent simulation closely resembling the feel of 8 Ball Pool.

Deterministic Multiplayer Architecture

Instead of replicating object transforms over the network, the game uses a deterministic simulation model.

  • When a player takes a shot, only the applied force data is sent to the server via RPC.
  • The server validates and forwards this input to the other client.
  • Each client runs the exact same physics simulation locally.
  • No continuous transform replication is used.

Once the physics simulation ends, all clients notify the server. The server then validates final ball positions to ensure consistency and prevent potential manipulation or cheating.

Because the physics simulation is deterministic and executed identically on all clients and the server, the results remain highly consistent while minimizing network bandwidth usage.
