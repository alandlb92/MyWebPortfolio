AutoBattle – Turn-Based Combat Simulator (C++)

AutoBattle is a console-based turn-based combat game and engine developed in modern C++, built as a study and challenge to engineer clean gameplay architecture, pattern application, and modular combat systems.

Project Overview

The core gameplay loop begins by configuring the battlefield dimensions, selecting a player class (e.g., Paladin, Warrior, Cleric, Archer), and procedurally selecting an enemy. The game then enters a turn loop where each character executes actions until one side is defeated.

Command Pattern Architecture

A major focus of this study was the integration of the Command Pattern to encapsulate character actions (Move, Regular Attack, Special Attack) as independent command objects. This pattern decouples gameplay logic from character data and allows easy extension, reordering, and sequencing of actions.

Commands such as SimpleAttackCommand, FireBallCommand, and WrathOfWarriorCommand inherit from a unified interface and are selected dynamically by the character’s turn system based on game state and configured rules.

Gameplay Components

  • GameManager – orchestrates initialization, dependency injection, and high-level flow
  • TurnSystem – manages turn order, resolves action execution, and updates state
  • InputHandle – processes console input and maps to in-game decisions
  • BattleField – represents the spatial grid and manages unit placement
  • Character and CharacterClass – define attributes and behavior bindings
  • Helpers – utility functions to abstract common operations

Technical Highlights

During development and refactoring, the following were achieved:

  • Command Pattern integration to encapsulate discrete character actions
  • Clean turn resolution via a dedicated TurnSystem
  • Decoupled input handling and game logic
  • Class hierarchy restructuring and separation of concerns
  • Clear grid and battlefield management logic
  • Modular action binding via data-driven configuration

Game Flow Logic

Each turn, the system:

  1. Evaluates available actions based on character stats
  2. Selects and queues the appropriate Command
  3. Executes the Command via the TurnSystem
  4. Updates state and resolves the next active entity

Design Considerations

The codebase was refactored to improve separation of concerns and reduce coupling. By moving logic out of monolithic structures and into dedicated Command classes, the system supports:

  • Action polymorphism and extensibility
  • Easy addition of new skills or behaviors
  • Isolated unit testing of game rules
  • Clear responsibility boundaries across subsystems

Conclusion

AutoBattle served as a valuable experiment in applying software engineering patterns to gameplay systems. By implementing the Command Pattern and reorganizing the architecture, I improved maintainability, modularity, and scalability of the game logic while gaining deeper insights into clean design in a C++ context.
