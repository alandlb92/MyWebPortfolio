Rendering & Engine Optimization

This project was already highly optimized, but as part of a large-scale production with a massive player base, further performance improvements and engine modernization were required.

I worked primarily on rendering systems across DirectX 11, PlayStation 4, and Metal, contributing to performance optimization and engine-level improvements.

GPU Benchmark System

I helped design and implement a runtime GPU benchmarking system. The game supported a wide range of graphics cards, and recommended settings were based on a predefined hardware list. However, many players used unsupported GPUs, leading to inaccurate configuration recommendations.

To address this, we developed a dedicated C++ benchmark project to evaluate GPU performance dynamically.

  • Created a standalone C++ benchmark module for GPU testing.
  • Developed three stress-test shaders:
    • Vertex Shader
    • Pixel Shader
    • Compute Shader
  • Designed shaders to execute heavy GPU workloads.
  • Measured execution time using GPU events to evaluate hardware performance.

The benchmark results were used to provide more accurate graphics presets for previously unsupported GPUs.

DX9 to DX11 Modernization

The game was originally built using DirectX 9, as it had more than 10 years of development history. Although a DX11 path existed (mainly for the Xbox One version), several legacy DX9 patterns were still in use.

One key modernization effort involved migrating shader parameter handling to proper Constant Buffers, replacing the older DX9-style individual parameter passing approach.

  • Analyzed and refactored legacy shaders.
  • Identified and grouped data into appropriate constant buffers.
  • Migrated parameter-based inputs to structured DX11 constant buffers.
  • Ensured compatibility across DirectX 11, PlayStation 4, and Metal.

This modernization improved maintainability, performance consistency, and cross-platform scalability.

Shader Compiler Improvements

I contributed to improvements in the engine’s shader compiler pipeline, particularly regarding shader variant management and precompiled shader packages.

  • Adapted the compiler to store constant data more efficiently.
  • Implemented and refined shader variant hashing logic.
  • Investigated and resolved duplicated shader variant generation.

After extensive debugging, I discovered an inconsistency in the shader variant hash generation logic. Fixing this issue eliminated duplicated variants, reduced the size of the precompiled shader package by several megabytes, and improved initial loading times.
