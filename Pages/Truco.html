Animations

The animations in this project were implemented using animation curves and coroutines, allowing precise timing control and flexible manipulation. This approach also enabled controlled randomization of final positions, creating more dynamic and natural card and UI movements.

Turn-Based System & Multiplayer Synchronization

I implemented the complete turn-based system for the Truco game, including turn flow management and interactive buttons for Truco requests.

  • Turn flow and game state management
  • Implementation of Truco request logic
  • Client synchronization during Truco calls

Special care was taken to ensure proper synchronization between clients, maintaining a consistent game state and preventing desynchronization issues.

Networking & Security

The game was built using a server-authoritative architecture to ensure fairness and security.

  • Card data exists exclusively on the server.
  • Clients do not have access to opponents' card information.
  • If an opponent leaves the scene, the hidden side of their cards remains empty.
  • Cards are revealed only through server-side RPC calls.

This architecture prevents client-side data inspection and ensures secure, fair multiplayer gameplay.
