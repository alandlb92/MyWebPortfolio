GAS Implementation

Uniqkiller originally used a custom ability and skill system that presented several architectural and replication issues, particularly when expanding the game with new abilities. Although a modular weapon system was already working well, the ability framework required a more scalable and network-reliable solution.

To address this, I led the migration to Unreal Engine’s Gameplay Ability System (GAS). Instead of rewriting everything at once, I adopted a smooth, incremental implementation approach, gradually replacing parts of the old system to minimize disruption.

I began by designing and implementing custom AttributeSets, establishing the foundation for the new attribute-driven architecture. After that, I integrated the system with the UI through proper bindings and implemented gameplay effects such as damage-over-time areas and healing zones. I also refactored the existing damage pipeline, adapting the TakeDamage logic to work with the new attribute-based system.

Implemented Abilities

  • Wall Shield – Allows the player to summon a defensive shield in front of them.
  • Search for Prey – Reveals enemy indicators on screen and applies a silhouette shader visible through walls.
  • Additional gameplay-driven skills built using GAS best practices.

Skill Tree & Progression System

I designed and developed the complete Skill Tree UI system, enabling players to unlock progression-based modifiers that directly modify values inside Unreal AttributeSets.

These modifiers were fully integrated with GAS through Gameplay Effects, dynamically applying attribute changes in a scalable and network-safe way, allowing players to customize and specialize their characters.

Additional Contributions

Beyond system architecture and UI implementation, I contributed to various gameplay improvements, bug fixes, and performance optimizations across the project.
