Engine Overview – RootEngine

RootEngine is a custom C++ real-time rendering engine built on top of DirectX 11. The architecture is modular and divided into core systems:

  • Rendering System (DX11 + HLSL)
  • Scene & Entity Structure
  • Mesh & SkeletalMesh Components
  • Animation System
  • Asset Conversion Pipeline
  • Input Subsystem (FaiaInputSystem)
  • Basic Editor (Viewport + Hierarchy)

Rendering System (DirectX 11)

The rendering layer was implemented directly using DirectX 11. The engine initializes device, swap chain, render target views, depth buffers, and manages the full rendering pipeline.

  • Custom HLSL Vertex and Pixel shaders
  • Manual constant buffer management
  • Simple textured mesh rendering
  • Global lighting implementation
  • Frame update and render separation

Models are rendered using a structured draw pipeline, binding vertex buffers, index buffers, shaders, textures, and constant buffers explicitly.

Mesh & Skeletal Architecture

The engine supports both static and skeletal meshes through a component-based design:

  • MeshComponent static geometry
  • SkeletalMeshComponent skinned meshes with bone data

Skeletal meshes include:

  • Bone hierarchy
  • Skinning weights
  • Bone transformation matrices sent to GPU
  • Animation playback system

Skinning is performed in the vertex shader using bone matrices updated per frame.

Asset Pipeline & Tooling

A full asset conversion workflow was implemented to avoid runtime parsing overhead.

  • FBX conversion using Assimp (C++)
  • Conversion to custom Mesh, SkinMesh and BoneInfo formats
  • PNG processing using LibPNG
  • JPEG processing using LibJPEG-Turbo
  • JSON serialization/deserialization via Nlohmann JSON

This pipeline allows preprocessing models and textures into optimized engine-native formats before runtime.

Input System – FaiaInputSystem

A dedicated input module was implemented to abstract Win32 Raw Input and provide engine-level bindings.

  • Raw Input registration (RegisterRawInputDevices)
  • Keyboard & mouse event translation
  • Resolution-normalized mouse axes
  • Event queue processing
  • Conditional bindings (modifier support)
  • Callback-based action and axis events

The system processes OS messages separately from the engine update loop, ensuring deterministic behavior and separation of concerns.

Scene & Data Structure (imgui)

The scene architecture defines entities composed of components. Internal structures support:

  • Transform hierarchy
  • Mesh attachment
  • Skeletal data binding
  • Render submission per frame

The scene graph structure is mirrored inside the editor hierarchy.

Editor Layer

A minimal editor layer was implemented on top of the engine runtime. It includes:

  • Viewport rendering inside the editor
  • Hierarchy panel
  • Basic scene inspection

The editor interacts directly with engine subsystems, sharing scene data structures.

Core Engineering Achievements

  • Custom DirectX 11 rendering pipeline
  • GPU skeletal animation implementation
  • Custom asset conversion tools
  • Native Win32 input abstraction
  • Scene graph architecture
  • Engine/editor integration

Conclusion

RootEngine represents a full-stack real-time rendering study, covering rendering, animation, asset processing, input abstraction, and tooling integration.

By implementing these systems from scratch, this project provided practical experience in low-level engine architecture, graphics programming, and modular subsystem design.
